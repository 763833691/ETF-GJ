# 04-ETF智能投教平台_前端开发实现_v0.1

> 版本：v0.1  
> 文档类型：前端开发实现说明  
> 适用阶段：竞赛 Demo 第一阶段 / 前端优先落地  
> 项目名称：ETF 智能投教平台  
> 核心定位：数据驱动、语义驱动、证据闭环的 ETF 智能投教平台  
> 部署约束：本阶段不使用 Docker。本地开发使用 Node.js + pnpm，服务器部署使用 Nginx + PM2 + Node.js。

---

## 0. 一句话结论

本阶段前端开发目标不是做一个临时 Demo 页面，而是快速搭建一个具有专业金融科技 SaaS 质感的产品雏形。

前端第一阶段必须完成：

```text
打开即用
无登录
专业 UI
液态玻璃风格
数据驱动语义画布
AI 投教方案卡片
场景推演
证据链报告
系统状态展示
```

第一版可以使用 `demo_seed` 数据，但所有数据必须明确标注来源，且前端结构必须为后续接入真实 ETF 数据接口、新闻数据接口、my_blueprint、Knowledge Hub、SWM 留出接口和组件边界。

## 1. 开发目标

### 1.1 本阶段目标

本阶段优先把前端效果做出来，满足竞赛演示和公网展示需要。

目标包括：

```text
1. 用户打开系统后直接进入主工作台。
2. 不出现登录、注册、用户中心、支付入口。
3. 主页面具备专业金融科技 SaaS 视觉效果。
4. 中间区域展示 ETF 语义世界画布。
5. 画布支持 React Flow 节点、连线、点击详情、类型筛选。
6. 右侧展示 AI 投教辅助方案。
7. 场景推演可点击切换并展示结果。
8. 证据链报告页可展示完整投教辅助报告。
9. 系统状态页展示数据源、模型状态、接口状态。
10. 后续可平滑接入真实 API。
```

### 1.2 本阶段不做

本阶段不做以下内容：

```text
1. 登录 / 注册 / 找回密码。
2. 用户权限和多租户切换界面。
3. 支付系统。
4. 自动交易。
5. 券商账户连接。
6. 真实下单能力。
7. 真实收益预测。
8. 复杂后台管理。
9. Docker 部署。
10. 复杂数据库建模。
11. 完整 my_blueprint / KH / SWM 真实接入。
```

## 4. 路由设计

### 4.1 路由表

```text
/               主工作台
/scenario       场景推演
/report         证据链报告
/system/swm     SWM 调试台，只读
/system/status  数据与模型状态，只读
```

### 4.2 路由要求

```text
1. 用户打开 / 后直接进入主工作台。
2. 不设置登录页。
3. 顶部导航在所有页面保持一致。
4. /system/swm 和 /system/status 是只读展示，不提供危险操作。
5. 任何页面都必须显示数据源状态或合规提示入口。
```

### 4.3 React Router 示例

```tsx
import { createBrowserRouter } from 'react-router-dom';
import { AppShell } from '../components/layout/AppShell';
import { WorkspacePage } from '../pages/WorkspacePage';
import { ScenarioPage } from '../pages/ScenarioPage';
import { ReportPage } from '../pages/ReportPage';
import { SystemSwmPage } from '../pages/SystemSwmPage';
import { SystemStatusPage } from '../pages/SystemStatusPage';
import { NotFoundPage } from '../pages/NotFoundPage';

export const router = createBrowserRouter([
  {
    path: '/',
    element: <AppShell />,
    children: [
      { index: true, element: <WorkspacePage /> },
      { path: 'scenario', element: <ScenarioPage /> },
      { path: 'report', element: <ReportPage /> },
      { path: 'system/swm', element: <SystemSwmPage /> },
      { path: 'system/status', element: <SystemStatusPage /> },
      { path: '*', element: <NotFoundPage /> },
    ],
  },
]);
```

## 7. SemanticWorldGraph 实现要求

### 7.1 组件职责

`SemanticWorldGraph` 负责将标准化语义图数据渲染为 React Flow 图谱。

输入：

```ts
export type SemanticWorldGraphProps = {
  nodes: SemanticGraphNode[];
  edges: SemanticGraphEdge[];
  selectedNodeId?: string;
  onNodeSelect?: (node: SemanticGraphNode) => void;
};
```

### 7.2 节点类型定义

```ts
export type SemanticObjectType =
  | 'UserProfile'
  | 'InvestmentGoal'
  | 'RiskProfile'
  | 'ETFProduct'
  | 'Index'
  | 'Sector'
  | 'Theme'
  | 'MarketMetric'
  | 'NewsEvent'
  | 'RiskSignal'
  | 'StrategyPlan'
  | 'EvidenceObject'
  | 'DataSource';

export type SemanticGraphNode = {
  id: string;
  type: 'semanticNode';
  position: { x: number; y: number };
  data: {
    objectType: SemanticObjectType;
    title: string;
    subtitle?: string;
    status?: 'active' | 'draft' | 'warning' | 'reviewed';
    confidence?: number;
    dataSource: 'demo_seed' | 'provider_api' | 'manual' | 'model_generated';
    updatedAt?: string;
    tags?: string[];
    metrics?: Record<string, string | number>;
  };
};
```

### 7.3 关系类型定义

```ts
export type SemanticRelationType =
  | 'has_goal'
  | 'has_risk_profile'
  | 'tracks'
  | 'belongs_to'
  | 'exposed_to'
  | 'has_metric'
  | 'affected_by'
  | 'increases_risk'
  | 'matches'
  | 'supported_by'
  | 'uses_data_source';

export type SemanticGraphEdge = {
  id: string;
  source: string;
  target: string;
  label: SemanticRelationType;
  animated?: boolean;
  data?: {
    description?: string;
    strength?: number;
    dataSource?: string;
  };
};
```

## 13. API Service 预留

第一阶段可以从 `demoSeed` 读取数据，但必须把数据访问封装在 service 层，避免后续重构。

### 13.1 api.ts

```ts
export const API_BASE_URL = import.meta.env.VITE_API_BASE_URL || '';

export async function request<T>(url: string, options?: RequestInit): Promise<T> {
  const response = await fetch(`${API_BASE_URL}${url}`, {
    headers: {
      'Content-Type': 'application/json',
      ...(options?.headers || {}),
    },
    ...options,
  });

  if (!response.ok) {
    throw new Error(`API request failed: ${response.status}`);
  }

  return response.json() as Promise<T>;
}
```

这不是普通 ETF 推荐页，
