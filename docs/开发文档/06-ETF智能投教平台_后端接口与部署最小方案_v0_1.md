# 06-ETF智能投教平台_后端接口与部署最小方案_v0.1

> 项目名称：ETF智能投教平台  
> 文档版本：v0.1  
> 文档定位：用于指导竞赛 Demo 第一阶段的后端最小接口、数据服务、模型调用、部署上线与验收。  
> 当前阶段：竞赛 Demo / 专业产品雏形  
> 关键约束：第一阶段 **不使用 Docker**。本地开发使用 Node.js + pnpm，服务器部署使用 Nginx + PM2 + Node.js + PostgreSQL 原生部署。  

---

## 0. 一句话结论

ETF智能投教平台第一阶段后端目标不是做完整金融数据平台，而是提供一个稳定、可部署、可演示、可扩展的最小服务层。

第一阶段后端需要支撑：

```text
前端打开即用
数据驱动语义画布
AI 投教辅助方案
场景推演
证据链报告
系统状态展示
公网部署演示
```

第一阶段可以使用 `demo_seed` 数据，但必须通过后端 API 输出，不能让前端大量写死业务数据。后端接口、数据结构和服务边界必须为后续接入真实 ETF 数据接口、新闻数据接口、PostgreSQL、my_blueprint、Knowledge Hub、SWM 留出空间。

## 1. 本阶段开发目标

### 1.1 必须实现

```text
1. 提供 Node.js + Express + TypeScript 后端服务。
2. 提供统一 API 前缀 /api。
3. 提供 /api/health 健康检查接口。
4. 提供 /api/demo/bootstrap 初始数据接口。
5. 提供 /api/semantic-world/graph 语义画布数据接口。
6. 提供 /api/profile/parse 用户画像解析接口。
7. 提供 /api/plan/generate AI 投教方案接口。
8. 提供 /api/scenario/simulate 场景推演接口。
9. 提供 /api/report/demo 证据链报告接口。
10. 提供 /api/system/status 系统状态接口。
11. 提供 demo_seed 数据文件和服务层。
12. 支持 OpenAI-compatible 模型配置预留。
13. 未配置模型时返回 503 model_not_configured，不伪造真实模型调用。
14. 支持 Nginx + PM2 + Node.js 原生部署。
15. 不使用 Docker，不生成 Dockerfile，不生成 docker-compose.yml。
```

### 1.2 暂不实现

```text
1. 登录注册。
2. 用户权限系统。
3. 支付系统。
4. 真实券商交易接口。
5. 自动交易。
6. 高并发行情推送。
7. 完整行情数据供应商接入。
8. 完整 my_blueprint / KH / SWM 真实联调。
9. 完整后台管理系统。
10. 多租户运营体系。
```

### 1.3 第一阶段部署目标

```text
1. 前端通过公网 IP 访问。
2. 后端监听 127.0.0.1:8000。
3. Nginx 托管前端 dist。
4. Nginx 将 /api 反向代理到 127.0.0.1:8000。
5. PM2 管理后端进程。
6. 服务器不使用 Docker。
7. 域名备案未完成前，优先使用公网 IP 演示。
```

## 4. 环境变量设计

### 4.1 .env.example

```env
# 基础配置
NODE_ENV=development
API_HOST=127.0.0.1
API_PORT=8000
PUBLIC_BASE_URL=http://127.0.0.1:5173

# 数据源模式
DATA_SOURCE_MODE=demo_seed
ENABLE_PROVIDER_API=false

# 数据库，第一阶段可以预留，P3 再强制使用
DATABASE_URL=postgresql://etf_user:your_password@127.0.0.1:5432/etf_swm_agent

# 模型配置，支持 OpenAI-compatible API
LLM_PROVIDER=openai_compatible
LLM_BASE_URL=
LLM_API_KEY=
LLM_MODEL=
LLM_ENABLED=false

# 安全配置
CORS_ORIGIN=http://127.0.0.1:5173
REQUEST_TIMEOUT_MS=30000
LOG_LEVEL=info
```

### 4.2 环境变量规则

```text
1. API Key 只能放在服务端 .env。
2. 前端不得展示完整 API Key。
3. 没有配置 LLM_API_KEY 或 LLM_ENABLED=false 时，AI 生成接口返回 503 model_not_configured。
4. DATA_SOURCE_MODE=demo_seed 时，前端必须显示“当前使用演示数据”。
5. ENABLE_PROVIDER_API=false 时，不得假装已经接入真实 ETF 数据接口。
```

## 7. Demo Seed 数据服务

### 7.1 demo_seed 的定位

`demo_seed` 是竞赛 Demo 第一阶段的数据源，不是伪造真实行情。

所有 demo_seed 数据必须满足：

```text
1. 明确标注 dataSource = demo_seed。
2. 前端明显显示“当前使用演示数据”。
3. 不伪装成实时行情。
4. 不写入正式 KH 或正式 SWM。
5. 后续可替换为 provider_api。
```

### 7.2 demoSeed.ts 内容

建议包含：

```text
1. etfProducts
2. indices
3. sectors
4. themes
5. marketMetrics
6. newsEvents
7. riskSignals
8. evidenceObjects
9. strategyPlans
10. semanticGraph
11. scenarioResults
12. reportDemo
```

### 7.3 demoSeed.service.ts

职责：

```text
1. 读取 demoSeed.ts。
2. 组合 bootstrap 数据。
3. 生成 semantic graph。
4. 根据 scenarioType 返回场景结果。
5. 提供报告数据。
```

## 8. 模型调用服务

### 8.1 model.service.ts

职责：

```text
1. 读取 LLM 配置。
2. 判断模型是否启用。
3. 调用 OpenAI-compatible API。
4. 记录最小调用日志。
5. 未配置时抛出 model_not_configured。
```

### 8.2 OpenAI-compatible 请求示例

```ts
const response = await fetch(`${baseUrl}/chat/completions`, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    Authorization: `Bearer ${apiKey}`,
  },
  body: JSON.stringify({
    model,
    messages,
    temperature: 0.4,
  }),
});
```

### 8.3 未配置模型的处理

```text
1. 后端返回 503 model_not_configured。
2. 前端显示模型未配置状态。
3. 不得用假 AI 结果冒充真实模型输出。
4. 可以展示“演示方案”，但必须标记 dataSource = demo_seed。
```

## 14. Nginx 配置

### 14.1 使用公网 IP 演示

Nginx 站点配置示例：

```nginx
server {
    listen 80;
    server_name _;

    root /www/wwwroot/etf-swm-agent/current/apps/web/dist;
    index index.html;

    client_max_body_size 20m;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 30s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    location ~* \.(js|css|png|jpg|jpeg|gif|svg|ico|webp)$ {
        expires 7d;
        add_header Cache-Control "public, max-age=604800";
        try_files $uri =404;
    }
}
```

重载 Nginx：

```bash
nginx -t
systemctl reload nginx
```

## 21. 最终结论

```text

清晰数据来源
```
