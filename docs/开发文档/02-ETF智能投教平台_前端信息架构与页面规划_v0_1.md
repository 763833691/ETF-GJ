## 0. 一句话结论

ETF 智能投教平台第一阶段前端应采用 **打开即用的专业金融科技工作台** 形态。

本阶段不做登录、不做注册、不做用户中心，用户访问公网链接后直接进入主工作台。页面重点不是普通表单和列表，而是围绕一个 **数据驱动的 ETF 语义世界画布** 展开：

```text
用户目标输入
→ 用户画像生成
→ ETF 数据 / 新闻事件 / 风险信号进入语义画布
→ AI 投教辅助方案生成
→ 场景推演
→ 证据链报告
→ 系统状态与数据来源说明
```

前端需要做到：

```text
1. 第一屏有专业金融科技产品质感。
2. 中间画布清晰体现“数据驱动 + 语义驱动”。
3. 页面流程能支撑 2-3 分钟竞赛演示。
4. 所有演示数据明确标注 demo_seed。
5. 所有方案均有合规提示，不输出买卖建议。
6. 前端组件命名保持通用，后续可适配 my_blueprint / KH / SWM。
```

## 1. 前端总体原则

### 1.1 打开即用

第一阶段前端不设计登录流程。

访问入口：

```text
/
```

用户打开后直接进入主工作台。

不出现：

```text
登录页
注册页
忘记密码
用户中心
租户切换
支付入口
API Key 明文配置入口
```

系统默认使用：

```text
demo_org
demo_project
demo_user
```

但这些默认身份不在用户界面中强调，只作为后端和数据结构预留。

---

### 1.2 页面少而精

第一阶段不追求页面数量，而追求演示闭环完整。

核心页面控制在 5 个：

```text
/
主工作台

/scenario
场景推演页

/report
证据链报告页

/system/swm
SWM 调试台，只读

/system/status
数据与模型状态页，只读
```

其中 `/` 是最重要页面，必须承载 70% 的演示内容。

---

### 1.3 主工作台优先

竞赛 Demo 的用户第一眼看到的不是营销官网，而是一个正在运行的专业平台。

因此首页不做传统 Landing Page，而是做 **Workspace 首页**：

```text
顶部状态栏
+ 产品标题
+ 用户目标输入
+ ETF 语义世界画布
+ AI 投教方案
+ 数据源状态
+ 证据链摘要
```

如需产品说明，可在主工作台顶部或左侧以简短说明卡片呈现，不单独做营销页。

---

### 1.4 数据驱动优先

中间画布必须体现数据来源和数据状态。

即使第一版使用 demo_seed，也要在视觉和交互上预留真实数据接口位置：

```text
ETF 数据状态
指数数据状态
新闻事件状态
风险信号状态
证据对象状态
最近更新时间
```

前端不得假装正在使用真实实时行情。

必须明确显示：

```text
当前数据源：demo_seed
真实 ETF 数据、指数数据和新闻数据接口已预留接入位
```

---

### 1.5 合规提示常驻

平台所有方案、推演、报告区域必须包含合规提示。

推荐全局提示语：

```text
本平台内容仅用于投资教育和辅助分析，不构成任何投资建议、收益承诺或交易指令。市场有风险，投资需谨慎。
```

在 UI 上可放置于：

```text
顶部状态栏右侧
AI 方案卡片底部
证据链报告底部
导出报告页脚
```

## 4. 主工作台页面 `/`

### 4.1 页面目标

主工作台是平台第一屏，承担以下目标：

```text
1. 让用户立刻知道这是 ETF 智能投教平台。
2. 让评委一眼看到“语义世界画布”。
3. 让用户输入投资目标并生成用户画像。
4. 展示 ETF 数据、新闻事件、风险信号如何进入画布。
5. 展示 AI 投教辅助方案。
6. 展示证据链和数据来源。
```

---

### 4.2 页面整体布局

推荐布局：三栏工作台。

```text
┌────────────────────────────────────────────────────────────┐
│ 顶部导航栏：产品名 / 数据源 / 模型状态 / 导出报告 / 免责声明 │
├─────────────────┬─────────────────────────┬────────────────┤
│ 左侧：目标与画像 │ 中间：ETF 语义世界画布   │ 右侧：AI 方案   │
│                 │                         │                │
│ 投资目标输入     │ 数据状态条               │ 方案卡片        │
│ 用户画像         │ React Flow 图谱           │ 风险提示        │
│ 偏好参数         │ 节点筛选 / 路径高亮       │ 证据摘要        │
│                 │                         │                │
├─────────────────┴─────────────────────────┴────────────────┤
│ 底部：新闻事件 / 风险信号 / 证据对象 / 最近动作摘要          │
└────────────────────────────────────────────────────────────┘
```

### 4.3 推荐宽度比例

```text
左侧 280-340px
中间 flex 1，最小 720px
右侧 360-420px
```

在较小屏幕下：

```text
左侧折叠为顶部输入卡
右侧方案区下移
中间画布保持优先展示
```

第一版可优先适配桌面端，不强求移动端。

### 6.4 节点类型

第一版图谱节点分为 6 类。

#### 用户类

```text
UserProfile
InvestmentGoal
RiskProfile
```

展示为蓝灰玻璃节点。

#### 资产类

```text
ETFProduct
Index
FundCompany
Sector
Theme
Holding
```

展示为低饱和青蓝节点。

#### 数据类

```text
MarketMetric
PerformanceMetric
LiquidityMetric
VolatilityMetric
DrawdownMetric
```

展示为银灰或淡蓝节点。

#### 事件类

```text
NewsEvent
PolicyEvent
MarketEvent
RiskEvent
Announcement
```

展示为淡琥珀或雾紫节点，不使用高饱和红色。

#### 方案类

```text
StrategyPlan
ScenarioRun
```

展示为低饱和绿色或淡紫节点。

#### 证据类

```text
EvidenceObject
DataSource
Citation
```

展示为白色玻璃节点，边框略强调。

## 9. 场景推演页面 `/scenario`

```text
当市场状态、新闻事件或用户条件发生变化时，ETF 语义世界和投教方案如何被重新解释。
```

### 9.2 页面布局

推荐布局：

```text
左侧：场景选择
中间：受影响语义路径
右侧：推演结果与风险解释
底部：方案稳定性对比
```

### 9.3 场景按钮

第一版场景：

```text
市场震荡
科技板块回撤
红利风格走强
投资周期缩短
风险偏好下降
流动性下降
新闻负面事件增加
政策利好行业主题
```

### 9.4 推演结果展示

每个场景点击后展示：

```text
受影响 ETF
受影响行业/主题
受影响风险因子
受影响关系路径
方案稳定性变化
需要重新评估的原因
证据来源
```

文案示例：

```text
在“科技板块回撤”场景下，科技主题 ETF 的波动风险权重上升。对于风险偏好中等且投资经验较低的用户，系统建议重新评估主题类 ETF 的配置比例。本说明仅用于投教分析，不构成买卖建议。
```

## 14. 前端交互流程

### 14.1 主流程

```text
用户打开 /
→ 系统加载 bootstrap 数据
→ 展示默认 demo_seed 图谱
→ 用户输入目标
→ 点击“生成 ETF 语义世界”
→ 左侧生成用户画像
→ 中间图谱高亮用户目标路径
→ 右侧更新 AI 投教方案
→ 底部更新风险信号和证据摘要
```

### 14.2 节点点击流程

```text
点击图谱节点
→ 打开 NodeDetailDrawer
→ 显示对象类型、数据来源、关系、证据、最近动作
→ 如果是证据节点，可跳转报告页对应证据
→ 如果是事件节点，可跳转场景推演页
```

### 14.3 场景推演流程

```text
进入 /scenario
→ 选择场景
→ 高亮受影响节点和关系
→ 展示风险解释
→ 展示方案稳定性变化
→ 提供返回工作台按钮
```

### 14.4 报告流程

```text
进入 /report
→ 自动汇总用户画像、方案、场景、证据
→ 显示风险提示和免责声明
→ 用户点击复制报告 / 导出 HTML
```

## 17. 给 Codex / Cursor 的开发约束

开发智能体必须遵守：

```text
1. 不要生成登录页。
2. 不要生成注册页。
3. 不要把项目做成普通 ETF 列表页。
4. 不要把中间画布做成静态图片。
5. 中间画布必须使用 React Flow / @xyflow/react。
6. 节点必须是低饱和液态玻璃风格。
7. 页面必须明确显示 data_source = demo_seed。
8. 不得输出买入、卖出、收益承诺等投资建议文案。
9. 所有方案必须显示免责声明。
10. 前端数据可以先用 demoSeed.ts，但字段要按真实接口结构设计。
11. 组件命名必须保持通用，不要写死成只能用于 ETF 的图谱组件。
12. 所有系统状态页面必须只读，不得暴露敏感配置。
```

## 18. 最终结论

ETF 智能投教平台第一阶段前端应聚焦“专业效果”和“完整演示路径”。

页面数量不要太多，核心是：

```text
一个主工作台
一个场景推演页
一个证据报告页
两个只读系统页
```

真正的产品亮点集中在主工作台中间的 **数据驱动 ETF 语义世界画布**。

它必须让用户和评委一眼看到：

```text
ETF 不是孤立数据条目，
而是与指数、行业、新闻、风险、用户目标和证据链共同组成一个可解释、可推演的语义世界。
```

这就是前端第一阶段的核心目标。
