# ETF 智能投教平台无容器部署说明

第一阶段部署路线：Node.js + pnpm + PM2 + Nginx。前端由 Nginx 托管静态构建产物，API 由 PM2 管理 Node.js 进程。

## 服务器准备

建议安装：

```text
Node.js 20 LTS 或更高版本
pnpm
PM2
Nginx
Git
```

## 目录建议

```text
/www/wwwroot/etf-swm-agent/current
```

前端产物：

```text
/www/wwwroot/etf-swm-agent/current/apps/web/dist
```

API 产物：

```text
/www/wwwroot/etf-swm-agent/current/apps/api/dist
```

## 首次部署

```bash
cd /www/wwwroot/etf-swm-agent/current
pnpm install --frozen-lockfile
pnpm --filter api build
pnpm --filter web build
cp .env.example .env
pm2 start ecosystem.config.cjs
pm2 save
```

将 `nginx/etf-swm-agent.conf` 放入 Nginx 站点配置目录后：

```bash
nginx -t
systemctl reload nginx
```

## 更新部署

```bash
cd /www/wwwroot/etf-swm-agent/current
git pull
pnpm install --frozen-lockfile
pnpm --filter api build
pnpm --filter web build
pm2 restart etf-swm-agent-api
nginx -t
systemctl reload nginx
```

也可以使用：

```bash
./scripts/deploy.sh
```

## 验证

```bash
curl http://127.0.0.1:8000/api/health
curl http://127.0.0.1:8000/api/system/status
curl http://127.0.0.1:8000/api/semantic-world/graph
```

浏览器访问服务器公网 IP，应直接进入主工作台，刷新 `/scenario`、`/report`、`/system/swm`、`/system/status` 不应返回 404。

## 安全边界

- 不在前端展示完整 API Key。
- 不开放模型 Key 编辑入口。
- 不开放数据删除、系统配置修改或真实数据导入入口。
- 当前数据源为 `demo_seed`，仅用于功能展示。
